# Release Notes `0.3.0`

- Date: 2026-03-03
- Type: stable release
- Scope: command model backend-agnostic + i18n advanced

## Highlights
- Nuovo command model pubblico in `dev.patric.commonlib.api.command`:
  - `CommandModel`, `CommandNode`, `CommandExecution`, `CommandContext`
  - `CommandValidator`, `ValidationIssue`, `CommandResult`
  - `CommandRegistry`, `CommandMetadata`, enum support (`ArgumentType`, `PermissionPolicy`, `ExecutionMode`)
- `CommandPort` redesign (breaking):
  - `register(CommandModel model)`
  - `unregister(String root)`
  - `supportsSuggestions()`
- Nuovo blocco i18n advanced in `dev.patric.commonlib.api.message`:
  - `MessageRequest`, `PlaceholderResolver`, `FallbackChain`, `PluralRules`
- Runtime defaults aggiornati:
  - `DefaultCommandRegistry`, `DefaultCommandValidator`
  - `AdvancedMiniMessageService`, `DefaultFallbackChain`, `DefaultPluralRules`

## Test & quality
- Contract tests command model.
- Validation and scheduler/guard integration tests per command pipeline.
- Test fallback locale multiplo, pluralizzazione `one/other`, resolver chain custom.
- Regressione suite core (`clean test javadoc build`, `check`) e consumer demo (`0.3.0` jar locale).

## Breaking changes
- `CommandPort` ha un nuovo contratto API (v0.2 callers da adattare).
- `MessageService` usa `render(MessageRequest request)` come entrypoint principale.

## Limiti espliciti
- Nessun adapter backend reale incluso (CommandAPI/Cloud restano backlog).
- Nessuna dipendenza esterna aggiunta al core.
- Modello embed-first invariato.
