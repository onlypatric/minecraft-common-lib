# Release Notes `0.8.0`

- Date: 2026-03-04
- Type: stable release
- Scope: adapter wave 1 (`CommandAPI`, `FastBoard`, `FancyHolograms`, `FancyNpcs`)

## Highlights
- Introdotto modello di binding adapter runtime-safe:
  - `PortBindingService`
  - delegating wrappers per `CommandPort`, `ScoreboardPort`, `HologramPort`, `NpcPort`
- Introdotti 4 moduli adapter in-repo:
  - `adapter-commandapi`
  - `adapter-fastboard`
  - `adapter-fancyholograms`
  - `adapter-fancynpcs`
- Capability detection robusta con reason codes standard e metadata backend.

## Runtime behavior
- Default runtime registra fallback no-op via delegating ports.
- Gli adapter bindano solo quando disponibili.
- In caso di binding failure o dependency non valida:
  - capability `unavailable(reason)`
  - delegate ripristinato al fallback no-op

## Build and quality gates
- Policy core invariata: nessuna dipendenza esterna hard nel modulo root.
- Nuova policy adapter: `verifyAdapterDependencyPolicy`.
- Nuovo source set/task opt-in:
  - `adapterIntegrationTest` (`-PrunAdapterIntegration=true`)

## Limitazioni
- Nessun adapter wave 2 in questa release (`Claims`, `Schematic`, `BossBar`, `GUI`, `Packets` restano backlog).
- Nessuna pubblicazione Maven remota inclusa nel ciclo `0.8.0`.
