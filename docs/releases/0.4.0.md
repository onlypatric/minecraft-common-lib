# Release Notes `0.4.0`

- Date: 2026-03-03
- Type: stable release
- Scope: GUI session layer adapter-friendly

## Highlights
- Nuovo package `api.gui` con modello session-oriented completo:
  - `GuiSessionService`, `GuiSession`, `GuiOpenRequest`, `GuiState`
  - `GuiEvent` (`GuiClickEvent`, `GuiCloseEvent`, `GuiTimeoutEvent`, `GuiDisconnectEvent`)
  - enum di supporto (`GuiCloseReason`, `GuiSessionStatus`, `ClickAction`, `GuiUpdateResult`, `GuiEventResult`)
- `GuiPort` ridisegnata in ottica adapter:
  - `open(GuiSession)`
  - `render(UUID, GuiState)`
  - `close(UUID, GuiCloseReason)`
  - `supportsPortableEvents()`
- Nuove implementazioni core:
  - `DefaultGuiSessionService`
  - `NoopGuiPort`
  - bridge policy `GuiPolicyRoutedEvent`
- Capability model esteso con `StandardCapabilities.GUI`.

## Runtime behavior
- Session timeout automatico con cancellazione task in close manuale.
- Close idempotente e race-safe.
- Eventi GUI instradati attraverso `EventRouter` e `PolicyHook`.

## Breaking changes
- `GuiPort` cambia contratto rispetto alle versioni precedenti (`open(UUID, String)` rimosso).

## Limiti espliciti
- Nessun adapter GUI reale incluso in `0.4.0`.
- Nessuna dipendenza esterna aggiunta al core.
- Modello embed-first invariato.
