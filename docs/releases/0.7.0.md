# Release Notes `0.7.0`

- Date: 2026-03-04
- Type: stable release
- Scope: arena/team/persistence foundation (core-only)

## Highlights
- Nuovo package `api.arena`:
  - `ArenaService`, `ArenaInstance`, `ArenaResetStrategy`, `ArenaResetContext`, `ArenaResetResult`
- Nuovo package `api.team`:
  - `TeamService` + `FriendlyFirePolicy`
  - `PartyService` con flow leader/invite/accept/kick/leave/disband
- Nuovo package `api.persistence`:
  - `YamlPersistencePort` + implementazione core `DefaultYamlPersistencePort`
  - `SqlPersistencePort` abstraction-only + `NoopSqlPersistencePort`
  - `SchemaMigrationService` integer-based + `DefaultSchemaMigrationService`

## Runtime wiring
`DefaultCommonRuntime` registra di default:
- `ArenaResetPort` (`NoopArenaResetPort`)
- `ArenaService`
- `TeamService`
- `PartyService`
- `YamlPersistencePort`
- `SqlPersistencePort`
- `SchemaMigrationService`

## Capability model
Aggiunte capability:
- `ARENA_RESET` -> `unavailable("No adapter installed")`
- `PERSISTENCE_YAML` -> `available("core-default")`
- `PERSISTENCE_SQL` -> `unavailable("No SQL adapter configured")`
- `TEAMS` -> `available("core-default")`
- `PARTIES` -> `available("core-default")`

## Quality and validation
- Test nuovi: arena lifecycle/reset strategy, team-party flow, persistence/migrations, benchmark harness.
- `integrationTest` source set aggiunto con harness opt-in.
- Build policy estesa (`verifyCoreDependencyPolicy`) ai package:
  - `api/arena`
  - `api/team`
  - `api/persistence`

## Compatibility
- Nessun breaking change sulle API match introdotte in `0.6.0`.
- Nessuna dipendenza esterna adapter introdotta nel core.
- SQL backend reale e stack arena/team/persistence avanzato restano evoluzione successiva (`0.8.0+`).
