# Release Notes `0.9.1`

- Date: 2026-03-04
- Type: stable patch release
- Scope: Paper Dialog API wrapper core-native

## Highlights
- Nuovo package `api.dialog` con modello esteso completo:
  - services: `DialogService`, `DialogTemplateRegistry`
  - session lifecycle: `DialogOpenRequest`, `DialogSession`, `DialogCloseReason`, `DialogEventResult`
  - events: `DialogSubmitEvent`, `DialogTimeoutEvent`, `DialogCloseEvent`
  - response: `DialogResponse`
  - template model: body/input/type/action specs tipizzati
- Runtime wiring default:
  - `DefaultDialogService`
  - `DefaultDialogTemplateRegistry`
  - `DialogPlayerLifecycleBridge`
- Policy integration:
  - submit/close/timeout passano da `EventRouter` tramite `DialogPolicyRoutedEvent`
- Capability:
  - `StandardCapabilities.DIALOG` pubblicata `available("core-default")` al bootstrap.

## Runtime behavior
- Session tracking deterministic-safe:
  - supporto `open/find/activeByPlayer/close/closeAll`.
  - timeout opzionale con scheduler.
  - gestione idempotente close concorrenti.
- Cleanup:
  - `QUIT` via player bridge.
  - `PLUGIN_DISABLE` in `DefaultCommonRuntime.onDisable()`.
- Degrade-safe:
  - in ambienti senza dialog backend pienamente disponibile, il wrapper non deve far fallire il plugin host.
  - capability viene marcata `unavailable("binding-failed:paper-dialog:<error>")` quando la backend call fallisce.

## Notes
- Nessuna dipendenza esterna nuova introdotta nel core.
- Nessun adapter separato richiesto per questa feature (wrapper core-native).
