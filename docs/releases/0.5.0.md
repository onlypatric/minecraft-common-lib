# Release Notes `0.5.0`

- Date: 2026-03-04
- Type: stable release
- Scope: HUD primitives (scoreboard session + bossbar service)

## Highlights
- Nuovo package `api.hud`:
  - `ScoreboardSessionService`, `ScoreboardSession`, `ScoreboardSnapshot`, `HudUpdatePolicy`
  - `BossBarService`, `BossBarSession`, `BossBarState`
  - enum/model shared (`HudAudienceCloseReason`, `HudBarColor`, `HudBarStyle`, ...)
- `ScoreboardPort` ridisegnata in modello session-oriented (breaking).
- Nuova `BossBarPort` adapter-facing.
- Nuovi no-op ufficiali:
  - `NoopScoreboardPort`
  - `NoopBossBarPort`
- Runtime wiring esteso con:
  - `DefaultScoreboardSessionService`
  - `DefaultBossBarService`
- Capability model esteso:
  - `StandardCapabilities.SCOREBOARD`
  - `StandardCapabilities.BOSSBAR`

## Behavior
- Policy default `competitiveDefaults`: 5 tick throttle, dedupe attivo, max 15 linee scoreboard.
- Cleanup audience su `QUIT` / `WORLD_CHANGE` / `PLUGIN_DISABLE`.

## Breaking changes
- `ScoreboardPort` non espone più `updateSidebar(UUID, String, List<String>)`.
- Nuovo contratto: `open(...)`, `render(...)`, `close(...)`.

## Limiti espliciti
- Nessun adapter esterno reale incluso in `0.5.0`.
- Nessuna dipendenza terza aggiunta nel core.
