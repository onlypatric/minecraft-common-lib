# Release Notes `0.6.0`

- Date: 2026-03-04
- Type: stable release
- Scope: reusable match/state engine baseline

## Highlights
- Nuovo package pubblico `api.match` con modello completo:
  - state lifecycle (`MatchState`, `MatchSessionStatus`, `MatchSession`)
  - policy (`MatchTimingPolicy`, `RejoinPolicy`, `MatchPolicy`)
  - orchestration API (`MatchEngineService`, `MatchCallbacks`, `MatchCleanup`)
  - operation results (`MatchTransitionResult`, `JoinResult`, `DisconnectResult`, `RejoinResult`)
- Nuovo runtime service `DefaultMatchEngineService` con loop deterministico single-engine e startup lazy.
- Bridge built-in eventi player (`quit/world-change`) per alimentare policy rejoin/timeout.
- Capability aggiunta: `StandardCapabilities.MATCH_ENGINE` (`available("core-default")`).

## Behavior
- Transizioni timer-driven supportate:
  - `COUNTDOWN -> RUNNING`
  - `RUNNING -> ENDING` (opzionale, timeout)
  - `ENDING -> RESET`
  - `RESET -> closed`
- Policy default competitive:
  - `rejoinWindowTicks=200`
  - `sessionTimeoutTicks=1200`

## Compatibilità
- Nessuna nuova dipendenza esterna nel core.
- Nessuna rottura sulle API preesistenti `0.5.0`.
- Arena/team/persistence restano fuori scope (`v0.7.0`).
