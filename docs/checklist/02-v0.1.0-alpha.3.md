# Checklist v0.1.0-alpha.3

## Obiettivo
Rafforzare UX da libreria comune (DX), mantenendo invariata l'architettura core.

## Checklist
- [ ] Introdurre linee guida ufficiali per scrivere `CommonComponent` production-ready.
- [ ] Definire standard logging runtime (prefix, livelli, eventi lifecycle).
- [ ] Aggiungere helper plugin-generic riusabili (es. guard, bootstrap utility, error wrappers).
- [ ] Definire policy di naming e package stability (`api` vs `internal`).
- [ ] Aggiungere checklist migrazione per plugin consumer legacy.
- [ ] Documentare pattern consigliato per load/enable/disable nei plugin host.

## Test obbligatori
- [ ] Contract tests su API pubbliche (`CommonRuntime`, `CommonScheduler`, `ServiceRegistry`).
- [ ] Test backward compatibility minima su API deprecate.
- [ ] Build completa verde (`clean test javadoc build`).

## Exit criteria
- [ ] DX docs complete e verificabili da un consumer nuovo.
- [ ] API `0.1.x` pronta per freeze RC.
