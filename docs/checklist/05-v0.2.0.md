# Checklist v0.2.0

## Obiettivo
Espandere il dominio plugin-generic: porte standard aggiuntive e contratti adapter-first.

## Checklist
- [ ] Definire nuove porte API plugin-generic (es. `NpcPort`, `HologramPort`, `ClaimsPort`, `SchematicPort`).
- [ ] Introdurre implementazioni no-op ufficiali per tutte le nuove porte.
- [ ] Definire capability detection standard per adapter opzionali.
- [ ] Documentare policy licenze per adapter GPL/proprietari.
- [ ] Aggiornare backlog adapter con priorità e criteri di adozione.

## Test obbligatori
- [ ] Contract tests su nuove porte e no-op behavior.
- [ ] Test di assenza dipendenze esterne nel core.

## Exit criteria
- [ ] API porte pronta per integrare adapter reali in versioni successive.
