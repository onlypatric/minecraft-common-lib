# Checklist v2.1.0 Module System

## Obiettivo
Introdurre un Module System first-class (`api.module`) con dependency graph deterministico, lifecycle orchestration soft-disable, testing ground interno e integrazione smoke su `SERVER-TEST`.

## Scope
- [x] Nuovo package pubblico `api.module` (`CommonModule`, `ModuleState`, `ModuleStatus`, `ModuleRegistry`, `ModulePlanResult`).
- [x] Estensione `CommonRuntime.Builder` con `module(...)`, `modules(...)`, `enableModuleDiagnostics(...)`.
- [x] Estensione `RuntimeBootstrap` con overload `build(plugin, modules)`.
- [x] Planner runtime deterministico (`ModuleGraphPlanner`) con missing/cycle detection.
- [x] Orchestrator lifecycle (`ModuleLifecycleOrchestrator`) con policy soft-disable.
- [x] Registry runtime (`DefaultModuleRegistry`) registrato in `ServiceRegistry`.
- [x] Coesistenza moduli + componenti legacy in `DefaultCommonRuntime`.
- [x] Testing ground interno `examples/module-playground`.
- [x] `SERVER-TEST` aggiornato con dump moduli `CLIB_SMOKE_MODULES` + report `modules.csv`.
