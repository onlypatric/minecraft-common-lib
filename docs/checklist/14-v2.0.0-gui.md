# Checklist v2.0.0 GUI Power-Up

## Obiettivo
Portare il layer GUI a un modello v2 tipizzato, semplice da usare, con DSL fluente, slot policies, integrazione dialog e primo adapter InvUI.

## Scope
- [x] Nuovo model API `api.gui` v2 (`GuiDefinition`, `SlotDefinition`, `GuiAction`, `GuiInteractionEvent`).
- [x] Nuovo authoring `GuiDsl` per chest 9x1..9x6.
- [x] Nuove action first-class: `ToggleStateAction`, `OpenSubMenuAction`, `BackMenuAction`.
- [x] Dialog state binding: `DialogResponseBinding` + estensione `DialogOpenOptionsMapping`.
- [x] Registry menu riusabili: `GuiDefinitionRegistry` + implementazione default runtime.
- [x] Redesign `GuiPort` adapter-facing su render model/patch + feature flags.
- [x] Runtime `GuiSessionService` con `interact(...)` e policy per slot.
- [x] Listener bridge Bukkit (`InventoryClick/Drag/Close`, `PlayerQuit`) integrato nel runtime.
- [x] Integrazione `DialogService` da action GUI (`OpenDialogAction`) con submit -> `GuiState`.
- [x] Modulo `adapter-invui` completato con backend InvUI reale (`Gui`/`Window`) e bind capability `GUI` supportato.
